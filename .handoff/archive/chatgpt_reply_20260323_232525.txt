	•	partially edited locally

	•	actually committed and pushed

From what we saw recently:
	•	some batches did commit and push at true stops
	•	but Batch G is currently partial and unfinished, so that kind of work is likely only local right now

So the answer is:
	•	Yes, Code is updating the repo at real stopping points
	•	No, it is not necessarily updating it continuously enough that you can assume GitHub always reflects the latest progress

Best rule of thumb:
	•	if terminal shows a commit hash + push to main, the repo is up to date for that cycle
	•	if terminal only shows edits / update lines / “Running Batch …”, then it may still just be local
